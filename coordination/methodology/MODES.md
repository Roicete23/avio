# Modos de uso

## 1. Modo descubrimiento

Usalo cuando no hay idea clara.

Objetivo:

- Generar opciones.
- Comparar nichos.
- Detectar energia personal.
- Evitar enamorarse de la primera idea.

Archivos clave:

- `PROMPT_FOR_CLAUDE_BLIND.md`
- `IDEA_SCOREBOARD.md`
- `DISAGREEMENTS.md`
- `IDEA_GRAVEYARD.md`

## 2. Modo validacion

Usalo cuando ya hay una idea candidata.

Objetivo:

- Definir usuario, dolor y promesa.
- Crear landing, mensajes y tracker.
- Medir senales antes de construir.

Archivos clave:

- `VALIDATION_PLAN.md`
- `OUTREACH_COPY.md`
- `USER_INTERVIEW_SCRIPT.md`
- `VALIDATION_TRACKER.csv`

## 3. Modo construccion

Usalo cuando la idea ya paso validacion.

Objetivo:

- Construir MVP pequeno.
- Mantener alcance cerrado.
- Verificar cada entrega.

Archivos clave:

- `ROADMAP.md`
- `WEEKLY_PLAN.md`
- `SCOPE_GUARDRAILS.md`
- `TEST_LOG.md`

## 4. Modo red team

Usalo antes de comprometerte con una idea o lanzamiento.

Objetivo:

- Buscar fallos.
- Detectar supuestos falsos.
- Definir senales de descarte.

Archivos clave:

- `RED_TEAM_REVIEW.md`
- `RISK_REGISTER.md`
- `DISAGREEMENTS.md`

## 5. Modo rescate

Usalo cuando el proyecto se atasca o pierde ilusion.

Objetivo:

- Reducir alcance.
- Recuperar el siguiente paso.
- Separar cansancio de mala idea.

Archivos clave:

- `PROJECT_TRUTHS.md`
- `SCOPE_GUARDRAILS.md`
- `BLOCKERS.md`
- `NEXT_ACTION.md`

## 6. Modo producto de metodologia

Usalo si queremos convertir este sistema en una herramienta propia.

Objetivo:

- Analizar si hay producto aqui.
- Prototipar una herramienta de coordinacion multi-IA.

Archivos clave:

- `FUTURE_PRODUCT_IDEA.md`
- `COMPETITOR_RESEARCH.md`
- `PROTOTYPE_PLAN.md`
