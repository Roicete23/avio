# Metodologia de cooperacion Codex-Claude

Esta carpeta guarda un metodo reutilizable para trabajar con varios modelos de IA usando el repositorio como memoria compartida.

La idea no es tener un ritual rigido. Es tener un sistema modular que se pueda activar segun el tipo de problema:

- descubrir ideas;
- validar producto;
- construir software;
- revisar estrategia;
- evitar dispersion;
- documentar decisiones;
- coordinar Codex, Claude, Gemini u otros modelos.

## Como usarla

1. Copia la carpeta `methodology/` a un proyecto nuevo.
2. Crea una carpeta `coordination/` en ese proyecto.
3. Elige un modo desde `MODES.md`.
4. Usa las plantillas de `templates/`.
5. Guarda decisiones, desacuerdos y descartes por escrito.

## Principio central

No buscamos que todos los modelos esten de acuerdo rapido. Buscamos criterio independiente primero, sintesis despues y ejecucion clara al final.

## Archivos principales

- `MASTER_PROMPT.md`: prompt maestro para iniciar la metodologia en otro chat.
- `MODES.md`: formas distintas de aplicar el sistema.
- `WORKFLOW.md`: flujo optimo recomendado.
- `FILESYSTEM_STRUCTURE.md`: estructura de carpetas y archivos.
- `INDEPENDENT_THINKING.md`: tecnicas para fomentar criterio independiente.
- `FUTURE_PRODUCT_IDEA.md`: como podria convertirse esta metodologia en producto propio.
