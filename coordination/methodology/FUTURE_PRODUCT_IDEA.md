# Idea independiente: herramienta de coordinacion multi-IA

Esta metodologia podria convertirse en un producto propio.

## Problema

Cada vez mas personas usan varios modelos de IA, pero la coordinacion entre ellos es manual, caotica y se pierde en chats.

Problemas:

- No hay memoria compartida clara.
- Los modelos se copian entre si si leen demasiado pronto.
- No hay control de decisiones.
- No hay red team sistematico.
- Las ideas se dispersan.
- El usuario actua como pegamento humano.

## Posible producto

Una herramienta que crea y mantiene un espacio de trabajo multi-IA:

- brief del proyecto;
- rondas ciegas;
- comparacion de respuestas;
- decisiones;
- desacuerdos;
- red team;
- scope guardrails;
- handoffs;
- roadmap;
- export a repo.

## Usuario posible

- founders solitarios;
- developers con IA;
- estudiantes haciendo proyectos;
- creadores que usan varios modelos;
- equipos pequenos que quieren pensar con IA sin perder control.

## MVP posible

Una app local o web donde:

1. El usuario escribe una idea.
2. La app genera prompts separados para Codex, Claude y Gemini.
3. El usuario pega respuestas.
4. La app sintetiza desacuerdos y decisiones.
5. Exporta una carpeta `coordination/`.

## Diferenciacion

No seria otro gestor de tareas. Seria una herramienta para:

- pensar mejor con varios modelos;
- preservar independencia;
- convertir conversaciones en decisiones;
- reducir dispersion.
