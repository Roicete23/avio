# Collaboration Protocol

## Principios

1. No asumir que la primera idea es la buena.
2. Separar validacion estrategica, diseno comercial y ejecucion web.
3. Registrar decisiones, desacuerdos y descartes.
4. Convertir cada decision en una tarea concreta.
5. Mantener el proyecto pequeno hasta tener senales reales del mercado.

## Flujo base

1. Entender objetivo y restricciones.
2. Formular hipotesis de cliente, problema, oferta y canal.
3. Pedir criterio independiente cuando haya incertidumbre fuerte.
4. Comparar respuestas y registrar desacuerdos.
5. Hacer red team de la idea finalista.
6. Elegir una version pequena y validable.
7. Crear landing/web funcional.
8. Contactar clientes reales y medir respuesta.
9. Ajustar oferta, precio, sector o canal.

## Regla de independencia

Cuando se pida criterio a Claude u otro modelo, debe responder primero sin leer la respuesta de Codex. Para eso se usara `PROMPT_FOR_CLAUDE_BLIND.md`.

## Definicion de avance

Una tarea avanza solo si produce algo verificable: archivo, decision escrita, web visible, lista de clientes, mensaje de venta, resultado de test o aprendizaje documentado.

## Definicion de validacion inicial

No basta con que la web se vea bien. La idea empieza a validarse cuando al menos algunos clientes potenciales hacen una de estas acciones:

- Piden precio.
- Preguntan condiciones.
- Solicitan catalogo.
- Aceptan una llamada.
- Hacen un pedido pequeno.
- Dan una objecion clara y repetida.
